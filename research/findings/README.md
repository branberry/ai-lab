# Findings

Writeups of what worked, what didn't, and what's next. Free-form markdown — one file per finding, named `YYYY-MM-DD-<slug>.md`.

A finding is the durable output of the loop. It should reference the `EXP-` and `TECH-` ids it summarizes and the `LL-` it ultimately addresses.

## Suggested structure

- **What I tried** — the technique(s) and the eval.
- **What happened** — baseline vs technique, with the numbers.
- **What it means** — does the weakness get measurably better? Are there new weaknesses introduced?
- **What's next** — the next `LL-` to probe or `TECH-` to try.
